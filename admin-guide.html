🛠️ 素材管理ガイド

Blog Material Hub の素材管理システムで、生成した画像を簡単に登録・公開できます。

📋 素材管理画面を開く

📌 概要

Blog Material Hub は 手動素材管理システム を採用しています。あなたが生成した画像を、メタデータ（タイトル、説明、タグ）と一緒に登録することで、自動的にギャラリーに反映されます。

🚀 クイックスタート

ステップ 1：管理画面を開く

素材管理画面 にアクセスします。

ステップ 2：画像をアップロード

生成した画像（PNG/JPEG/WebP）をドラッグ&ドロップ、または「ファイルを選択」でアップロードします。

ステップ 3：メタデータを入力
  • 📝 タイトル：素材の名前（例：「観葉植物のある部屋」）
  • 📄 説明文：どんなブログで使えるか、活躍シーン
  • 🏷️ タグ：検索用のキーワード（複数可）
  • 📂 カテゴリ：素材の種類を選択
  • 🤖 生成プロンプト：オプション
ステップ 4：JSON生成

【登録用JSON生成】ボタンをクリック。管理画面下部に JSON 形式のデータが表示されます。

ステップ 5：JSON をコピー

【JSONをコピー】ボタンで、生成されたデータをコピーします。

ステップ 6：data.json に追記

GitHub 上で data.json を開き、コピーしたデータを "materials" 配列に追記します。

ステップ 7：git push で反映

GitHub に push すると、自動的にギャラリーに反映されます。

🎯 カテゴリ一覧

  • 🌿 plants：観葉植物、花、自然風景など
  • 🐾 animals：動物、生き物など
  • 🌅 bg：背景素材、風景など
  • ☕ daily：日常生活、日用品など
  • 📱 gadget：ガジェット、テック製品など
  • 🎨 material：デザイン要素、素材など

💡 タグの付け方のコツ

  • 検索しやすいキーワードを使う（例：「猫」「かわいい」「動物」）
  • 複数タグを活用（3-5個が目安）
  • カテゴリとタグが重複してもOK
  • 汎用的なタグと具体的なタグを混ぜる

⚙️ 上級：Python スクリプトを使用

対話モード

コマンドラインで対話的に素材を追加：

python3 add_material.py

質問に従って入力すると、自動的に data.json に追記されます。

JSON 形式で追加

JSON 形式で素材情報を直接入力：

python3 add_material.py --json

JSON 形式の例：

{ "title": "観葉植物のある部屋", "description": "インテリアとしての観葉植物を...", "tags": ["植物", "インテリア", "自然"], "category": "plants", "prompt": "A cozy room with potted plants..." }

バッチ追加

複数の素材を一括追加：

python3 add_material.py --batch materials.json

materials.json は以下の形式：

{ "materials": [ { "title": "素材1", ... }, { "title": "素材2", ... } ] }

📊 data.json の構造

{ "materials": [ { "id": 1, "title": "素材タイトル", "description": "説明文", "tags": ["タグ1", "タグ2"], "category": "plants", "prompt": "プロンプト", "downloadCount": 0, "createdDate": "2026-03-24" } ], "metadata": { "totalMaterials": 1, "lastUpdated": "2026-03-24", "updateFrequency": "daily", "categories": [...] } }

🔄 ワークフロー例

  1. 管理画面 を開く
  2. 画像をアップロード

  5. ギャラリーに反映

🔐 セキュリティ上の注意

  • data.json は公開されます：プライベート情報を含めないでください
  • 画像は GitHub にアップロード：大きなファイルは避けてください（推奨：1-5MB）
  • 著作権に注意：利用規約に従って、著作権フリーの素材を使用

❓ よくある質問

Q. 素材を削除したいのですが？

data.json からその素材を削除して git push してください。ギャラリーから消えます。

Q. 素材を編集したいのですが？

data.json 上の該当素材の情報を直接編集して git push します。

Q. ダウンロード数がリセットされます

素材の編集時に downloadCount フィールドを保持してください。削除してしまうと 0 にリセットされます。

Q. 画像をどこに保存すれば？

管理画面では、画像は Base64 形式で JSON に変換されます。実際の画像は GitHub にアップロードしてください（別フォルダに）。

🎓 ベストプラクティス

  • 毎日継続：毎日数個の素材を追加すると、SEO的にも効果的
  • 同じフォーマット：説明文のスタイルを統一
  • タグの一貫性：同じタグを繰り返し使用
  • 画像品質：高品質な画像は DL 数が増える傾向
