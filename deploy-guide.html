🚀 デプロイガイド

Blog Material Hub を GitHub Pages で公開するための手順です。

前提条件

  • Git がインストール済み
  • GitHub アカウント
  • ローカルの blog-material-hub フォルダ

ステップ 1：GitHub リポジトリを作成

GitHub にログイン → 新しいリポジトリを作成

https://github.com/new

  • Repository name: blog-material-hub
  • Description: ブログ素材配信サイト
  • Public / Private: Public（公開）
  • Initialize: チェックなし（後で push するため）

✅【Create repository】ボタンをクリック

ステップ 2：ローカルで Git を初期化

ターミナル/コマンドプロンプトで blog-material-hub フォルダに移動：

cd blog-material-hub

Git を初期化：

git init

GitHub リポジトリを remote として追加（YOUR_USERNAME は自分のGitHubユーザー名に変更）：

git remote add origin https://github.com/YOUR_USERNAME/blog-material-hub.git

例：

git remote add origin https://github.com/hajime085/blog-material-hub.git

ステップ 3：ファイルをステージング

すべてのファイルを追加：

git add .

確認：

git status

追加されたファイル一覧が表示されます。

ステップ 4：コミット

git commit -m "Initial commit: Blog Material Hub setup"

ステップ 5：GitHub に Push

git branch -M main git push -u origin main
💡 注： main ブランチがデフォルトブランチになっていることを確認してください。

ステップ 6：GitHub Pages を有効化

GitHub リポジトリページ → Settings → Pages
  1. Source セクションで「Deploy from a branch」を選択
  2. Branch を「main」に設定
  3. Folder を「/(root)」に設定
  4. ✅【Save】ボタンをクリック
✅ 完成！
数秒から1分で、以下の URL でサイトが公開されます：
https://YOUR_USERNAME.github.io/blog-material-hub/

ステップ 7：カスタムドメイン設定（オプション）

Cloudflare を使用する場合

Cloudflare ダッシュボード → DNS
  1. 新しいレコードを追加
  2. Type: CNAME
  3. Name: material（またはサブドメイン）
  4. Content: YOUR_USERNAME.github.io
  5. 例：hajime085.github.io
  6. ✅ Save

その後、GitHub リポジトリで：

  1. Settings → Pages → Custom domain
  2. ドメインを入力（例：material.example.com）
  3. ✅ Save
💡 注： DNS の反映に 24-48 時間かかる場合があります。

📝 今後の更新方法

素材を追加したら：

git add . git commit -m "Add new materials" git push

自動的に GitHub Pages が更新されます。

🔍 デプロイ状態の確認

GitHub リポジトリ → Actions

デプロイが進行中 / 完了したかを確認できます。

✅ グリーンチェックが表示されたら成功です。

❓ トラブルシューティング

Q. ページが表示されない

確認項目：

  • GitHub リポジトリが Public（公開）になっているか
  • GitHub Pages が有効になっているか（Settings → Pages）
  • ブランチが「main」に設定されているか
  • Folder が「/(root)」に設定されているか

Q. CSS が効かない / ページが崩れている

原因： ブラウザのキャッシュ

対策： Ctrl + Shift + Delete で キャッシュをクリア

Q. CNAME が機能しない

確認：

  • CNAME ファイルが GitHub に push されているか
  • CNAME 内容が正しいドメインか
  • DNS レコード（Cloudflare 等）が正しく設定されているか
  • DNS の反映を待つ（24-48 時間）
